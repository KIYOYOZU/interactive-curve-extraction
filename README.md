# 曲线提取工具

本工具用于从包含多个子图的论文图片中，交互式地提取目标子图的曲线数据。当前针对 `图片1.png` 适配，该图片包含 5 个上下排列的子图，目标是**从上往下数第二个子图**，提取其中的三组数据：黑色空心圆实验点、蓝色曲线、红色曲线。

提取完成后生成以下产物：

| 产物 | 说明 |
|------|------|
| `图片1_提取结果.dat` | 坐标轴标定信息，供下次运行自动复用 |
| `图片1_提取结果.xlsx` | 提取的三组曲线数值数据 |
| `图片1_提取结果对比.png` | Excel 数据与原图裁剪区域的视觉对比图 |
| `logs.txt` | 完整交互日志，可用于在 Excel 导出失败时恢复数据 |

---

## 快速开始

### 环境依赖

建议在 Windows Python 环境下运行。必需依赖：

```powershell
python -m pip install numpy pandas matplotlib openpyxl
```

`xlsxwriter` 为备用 Excel 引擎，`openpyxl` 缺失时主脚本会自动回退到它；`scipy` 为可选依赖，仅用于对比图中蓝色和红色曲线的样条平滑显示，缺失时退化为线性插值。建议一并安装：

```powershell
python -m pip install numpy pandas matplotlib openpyxl xlsxwriter scipy
```

### 标准运行流程

**第一步：交互提取**

```powershell
python interactive_extract_second_plot.py
```

首次运行时，程序会打开原图窗口，引导用户依次点击 4 个坐标轴参考点，并在控制台输入对应真实值，完成坐标轴标定。标定信息随即写入 `图片1_提取结果.dat`，后续运行会自动加载并直接进入曲线采点阶段，无需重复操作。如需重新标定，删除该 `.dat` 文件后重新运行即可。

**第二步（可选）：从日志恢复 Excel**

若交互已完成但 Excel 因环境问题未能正常写出，可运行：

```powershell
python recover_excel_from_logs.py
```

脚本会读取 `logs.txt` 和 `图片1_提取结果.dat`，重建并导出 Excel，自动处理日志中的撤销记录，只保留最终有效点。

**第三步（可选）：生成对比图**

```powershell
python plot_excel_vs_cropped_image.py

# 同时弹出预览窗口
python plot_excel_vs_cropped_image.py --show
```

### 手动指定文件路径

三个脚本均支持通过命令行参数手动指定输入输出文件：

```powershell
# 主脚本：指定图片路径
python interactive_extract_second_plot.py "C:\path\to\图片1.png"

# 恢复脚本：指定日志、DAT、输出 Excel 路径
python recover_excel_from_logs.py "C:\path\to\logs.txt" "C:\path\to\图片1_提取结果.dat" "C:\path\to\output.xlsx"

# 对比图脚本：指定 Excel、DAT、原图、输出图路径
python plot_excel_vs_cropped_image.py "C:\path\to\图片1_提取结果.xlsx" "C:\path\to\图片1_提取结果.dat" "C:\path\to\图片1.png" "C:\path\to\output.png"
```

## 交互流程详解

### 第一步：坐标轴标定

程序显示原图后，提示用户依次点击以下 4 个参考点：

1. X 轴起点（对应某个已知 x 真实值）
2. X 轴终点（对应某个已知 x 真实值）
3. Y 轴起点（对应某个已知 y 真实值）
4. Y 轴终点（对应某个已知 y 真实值）

点击完成后，程序在控制台依次询问这 4 个点的真实值，完成像素坐标到真实坐标的映射关系建立。注意：用户点击的 Y 方向两个参考点是**中间刻度**，对应真实值 −20 和 20，并非子图上下边界；程序会自动将显示范围向上下各扩展半个区间，最终裁剪并显示到 −40 至 40 的完整范围。

标定信息立即写入 `.dat` 文件，即使后续操作中断，下次运行也可自动恢复。

### 第二步：曲线采点

进入采点阶段后，程序在目标子图的标定 x 范围内均匀生成 **40 条等间距竖线**作为辅助。采点顺序固定为：**黑色实验数据 → 蓝色曲线 → 红色曲线**。

| 鼠标操作 | 效果 |
|----------|------|
| 左键单击 | 记录当前高亮竖线与曲线的交点 |
| 右键单击一次 | 撤销当前曲线的最后一个点 |
| 连续右击两次 | 当前曲线采满后，切换到下一条曲线 |

单右键与连续双右键通过 500 ms 延时计时器自动区分，避免在切换时误删最后一个有效点。

当前采样规则：

- **黑色实验点**：取第 3 到第 38 条竖线，共 **36 个点**（去掉最左侧 2 条和最右侧 2 条）
- **蓝色曲线**：取全部 40 条竖线，共 **40 个点**
- **红色曲线**：取全部 40 条竖线，共 **40 个点**

### x 坐标的处理方式

采点时，用户鼠标只负责确定 **y 方向**的位置。**x 坐标**不使用鼠标的实际横向像素位置，而是在点击瞬间直接绑定到当前高亮竖线对应的均匀划分 x 值。因此 `logs.txt` 中打印的 `x = ...` 以及 Excel 中的 x 列，均已是修正后的均匀采样位置，不需要任何后处理再修正。

## 输出数据格式

### DAT 标定文件（`图片1_提取结果.dat`）

保存坐标轴 4 个参考点的完整信息，每条记录包含：标记名（`x_start` / `x_end` / `y_start` / `y_end`）、像素 x、像素 y、对应真实值。后续运行主脚本时优先读取该文件，自动跳过标定步骤；日志恢复脚本也依赖该文件重建坐标系。

### Excel 文件（`图片1_提取结果.xlsx`）

包含以下 4 个工作表：

| 工作表 | 内容 |
|--------|------|
| `data` | 主数据表，第一列为 x，后三列为 `black` / `blue` / `red` 的 y 值，共 40 行 |
| `x_values` | 单独保存 40 个均匀采样 x 值 |
| `calibration` | 4 个标定点信息 |
| `meta` | 记录本次使用的日志文件路径 |

`data` 表中黑色实验点仅有 36 个有效值，第 1、2 行和第 39、40 行的 `black` 列为空，以保持与统一 x 网格的对齐。示例：

| x | black | blue | red |
|---|-------|------|-----|
| −40.000000 | NaN | −2.288230 | −14.301436 |
| −37.948718 | NaN | −0.858086 | −12.299235 |
| −35.897436 | 0.286029 | 0.572057 | −12.013206 |

### 对比图（`图片1_提取结果对比.png`）

上子图绘制从 Excel 读取的三组曲线数据（黑色实验点使用放大的空心圆，蓝色和红色使用样条平滑插值）；下子图显示从原图按 y 扩展规则裁剪出的目标子图区域。两个子图共用同一套 x 范围，下图保持原图裁剪区域的像素宽高比，避免因 x/y 像素缩放不一致导致的变形。

### 日志文件（`logs.txt`）

主脚本运行时自动将控制台输出同步写入该文件，记录：是否加载了已有 DAT、当前采样规则、每个点对应的竖线编号、修正后的 x 值、y 值、以及所有撤销操作。这是日志恢复脚本能重建 Excel 的基础。

## 常见问题

**黑色列为什么有空值？**
当前配置下总共 40 个 x 采样位，黑色实验点仅采第 3 到第 38 条竖线共 36 个点，因此 `data` 表中黑色列首尾各 2 行为空，用于与统一 x 网格对齐。这是预期行为。

**为什么重新运行时没有提示点击 4 个标定点？**
程序检测到 `图片1_提取结果.dat` 后，自动加载已有标定信息并跳过标定步骤。如需重新标定，删除该文件后再运行主脚本即可。

**Excel 导出失败怎么办？**
先检查是否缺少 `openpyxl` 和 `xlsxwriter`（主脚本支持二选一自动回退）。若交互已结束但 Excel 仍未生成，直接运行 `recover_excel_from_logs.py` 从日志重建。

**对比图的下子图为什么不按真实坐标等比例显示？**
该图的 x 与 y 像素缩放不同。若按真实坐标等比例显示，下子图会产生明显压缩变形，与原图视觉不一致。当前版本改为按原图裁剪区域的像素宽高比保持显示比例。

# 曲线提取工具说明

## 1. 项目简介

本目录用于对单张论文图片中的目标子图进行交互式曲线数据提取，并生成：

- `Excel` 数据文件
- `DAT` 标定文件
- `logs.txt` 交互日志
- 数据曲线与原图裁剪区域的对比图

当前这套工具针对本目录中的 `图片1.png` 已完成适配，目标是提取从上往下数第二个子图中的三组数据：

- 黑色实验点
- 蓝色曲线
- 红色曲线

当前采样规则如下：

- 总共生成 `40` 条等间距竖线
- 黑色实验点取中间 `36` 个点，对应第 `3` 到第 `38` 条竖线
- 蓝色曲线取 `40` 个点
- 红色曲线取 `40` 个点

## 2. 目录文件说明

- `图片1.png`
  原始图片。

- `interactive_extract_second_plot.py`
  主交互脚本。用于标定坐标轴、交互选点、自动记录日志、导出 Excel 和 DAT。

- `recover_excel_from_logs.py`
  日志恢复脚本。用于在交互完成但 Excel 导出失败时，从 `logs.txt` 和 `.dat` 中恢复有效数据并重新生成 Excel。

- `plot_excel_vs_cropped_image.py`
  对比图脚本。上子图显示 Excel 中读取到的三组数据，下子图显示按步骤 2 的规则裁剪出的原图区域。

- `图片1_提取结果.dat`
  标定信息文件。保存坐标轴 4 个参考点的像素坐标与真实值。后续运行主脚本时会优先读取它，自动跳过标定步骤。

- `图片1_提取结果.xlsx`
  当前提取结果。`data` 工作表第一列为 `x`，后 3 列为 `black / blue / red`。

- `图片1_提取结果对比.png`
  Excel 数据与裁剪原图的对比图。

- `logs.txt`
  交互运行日志。现在由主脚本自动生成，不需要再手工复制终端内容。

- `task_plan.md`
  任务计划记录。

- `findings.md`
  过程发现记录。

- `progress.md`
  进度与测试记录。

## 3. 环境依赖

建议使用当前已验证的 Windows Python 环境。

必需依赖：

- `numpy`
- `pandas`
- `matplotlib`

Excel 导出依赖：

- 优先使用 `openpyxl`
- 若缺少 `openpyxl`，主脚本会自动尝试 `xlsxwriter`

可选依赖：

- `scipy`
  仅用于对比图中蓝线和红线的平滑显示；没有也能运行，只是曲线会退化为线性加密插值。

安装示例：

```powershell
python -m pip install numpy pandas matplotlib openpyxl xlsxwriter scipy
```

## 4. 主流程说明

### 4.1 第一次运行：交互提取

运行：

```powershell
python interactive_extract_second_plot.py
```

首次运行时，如果目录中不存在 `图片1_提取结果.dat`，流程如下：

1. 打开原图窗口。
2. 依次点击 4 个标定点：
   - `X 轴起点`
   - `X 轴终点`
   - `Y 轴起点`
   - `Y 轴终点`
3. 在控制台输入这 4 个点对应的真实值。
4. 程序立即把标定信息写入 `图片1_提取结果.dat`。
5. 进入曲线选点阶段。

### 4.2 后续运行：自动跳过标定

如果目录中已经存在 `图片1_提取结果.dat`，再次运行主脚本时会：

- 自动读取已有标定信息
- 直接跳过第 1 步坐标轴参考点选择
- 直接进入曲线提取

如果需要重新标定，删除 `图片1_提取结果.dat` 后再运行主脚本即可。

## 5. 曲线选点规则

主脚本进入第二步后，会在目标子图上生成 `40` 条等间距竖线。

选点顺序固定为：

1. 黑色实验数据
2. 蓝色曲线
3. 红色曲线

操作规则：

- `左键`
  记录当前高亮竖线上的交点。

- `右键一次`
  撤销当前曲线的上一个点。

- `连续右击两次`
  当且仅当当前曲线已经采满时，进入下一条曲线。

当前点数规则：

- 黑色实验点：
  仅取第 `3` 到第 `38` 条竖线，共 `36` 个点。

- 蓝色曲线：
  取第 `1` 到第 `40` 条竖线，共 `40` 个点。

- 红色曲线：
  取第 `1` 到第 `40` 条竖线，共 `40` 个点。

## 6. 关于 x 坐标修正

这是本项目中非常重要的一点。

选点时：

- 用户鼠标实际只负责点击 `y` 方向的位置
- `x` 并不是使用鼠标点击时的横坐标
- `x` 会在记录当下直接绑定到当前那条等间距竖线对应的 `x` 值

也就是说，`x` 坐标不是后处理时才修正，而是在选点时就已经固定为“均匀划分竖线的值”。

因此：

- `logs.txt` 中打印的 `x = ...` 已经是修正后的结果
- `Excel` 中的 `x` 也是同一套均匀竖线的值

## 7. 关于 y 方向裁剪

步骤 2 中的目标子图裁剪，不是直接拿 `Y 轴起点` 和 `Y 轴终点` 的像素位置作为上下边界。

原因是：

- 用户点击的两个 `Y` 参考点是中间刻度
- 它们对应的是 `-20` 和 `20`
- 但实际需要显示的整个子图范围应扩展到 `-40` 到 `40`

当前程序采用的规则是：

1. 读取 `Y` 标定真实值区间
2. 计算该区间的半跨度
3. 在上下各扩展半个区间
4. 再把扩展后的真实值范围反算回像素边界

因此当前例子中：

- `-20 ~ 20`
  会自动扩成
- `-40 ~ 40`

这套规则也被 `plot_excel_vs_cropped_image.py` 复用，用来生成对比图。

## 8. 输出文件说明

### 8.1 DAT 文件

文件：

- `图片1_提取结果.dat`

用途：

- 保存 4 个标定点
- 作为后续自动跳过标定的依据
- 作为日志恢复时的坐标参考

内容包括：

- `x_start`
- `x_end`
- `y_start`
- `y_end`

每条记录包含：

- 标记名
- 像素 `x`
- 像素 `y`
- 真实值

### 8.2 Excel 文件

文件：

- `图片1_提取结果.xlsx`

工作表说明：

- `data`
  第一列为 `x`，后 3 列为 `black / blue / red`

- `x_values`
  单独保存整套 `40` 个均匀采样 `x`

- `calibration`
  保存标定点信息

- `meta`
  保存本次恢复所使用的日志路径

当前 `data` 表的结构示例：

| x | black | blue | red |
|---|---|---|---|
| -40.000000 | NaN | -2.288230 | -14.301436 |
| -37.948718 | NaN | -0.858086 | -12.299235 |
| -35.897436 | 0.286029 | 0.572057 | -12.013206 |

说明：

- 黑色实验点当前只有 `36` 个，因此在 `40` 行总表中首尾会留空
- 蓝色和红色都有 `40` 个有效点

### 8.3 日志文件

文件：

- `logs.txt`

当前主脚本会自动把控制台输出同步写入该文件。

日志中会记录：

- 是否自动加载了已有 DAT
- 当前采样规则
- 每个点对应的竖线编号
- 每个点的修正后 `x`
- 每个点的 `y`
- 撤销操作

这也是 `recover_excel_from_logs.py` 能恢复 Excel 的基础。

## 9. 日志恢复脚本

运行：

```powershell
python recover_excel_from_logs.py
```

默认读取：

- `logs.txt`
- `图片1_提取结果.dat`

默认输出：

- `图片1_提取结果.xlsx`

用途：

- 当交互已经完成，但由于环境问题未能成功写出 Excel 时使用
- 当前脚本会自动处理日志中的撤销动作
- 只保留最终有效点

恢复逻辑：

1. 从日志中读取每个“记录点”事件
2. 遇到“已撤销上一点”就回退最后一个点
3. 按竖线编号把结果重新对齐回 `40` 个 `x`
4. 根据 DAT 中的标定值生成均匀 `x`
5. 导出为新的 Excel

## 10. 对比图脚本

运行：

```powershell
python plot_excel_vs_cropped_image.py
```

默认读取：

- `图片1_提取结果.xlsx`
- `图片1_提取结果.dat`
- `图片1.png`

默认输出：

- `图片1_提取结果对比.png`

图像结构：

- 上子图：
  读取 Excel 中的三组数据并绘图

- 下子图：
  按步骤 2 的裁剪规则，从原图中截取目标区域

当前绘图特性：

- 黑色实验点使用更大的空心圆显示
- 蓝色和红色曲线使用更平滑的样条插值绘制
- 上下子图共用同一套 `x` 范围
- 显示比例按原图裁剪区域的像素宽高比保持，不再压缩变形

如果需要弹出窗口同时查看，可以使用：

```powershell
python plot_excel_vs_cropped_image.py --show
```

## 12. 常见问题

### 12.1 为什么黑色列有空值

因为当前配置下：

- 总共 `40` 个 `x`
- 黑色实验点只有 `36` 个

所以黑色列会在前 2 行和后 2 行为空，用于和统一的 `x` 网格对齐。

### 12.2 为什么重新运行时没有让我点 4 个标定点

因为程序检测到了 `图片1_提取结果.dat`，自动加载了已有标定信息。

如需重新标定，请删除该文件后再运行主脚本。

### 12.3 如果 Excel 导出失败怎么办

先看是否缺少 Excel 引擎依赖。

当前主脚本已支持：

- `openpyxl`
- `xlsxwriter`

如果交互已经结束但 Excel 仍未生成，可以直接运行：

```powershell
python recover_excel_from_logs.py
```

### 12.4 对比图为什么要按原图比例显示

因为本图的 `x/y` 像素缩放并不相同。

如果强行按“真实坐标等比例”去显示，下图会发生明显压缩变形，和原图不一致。当前版本已经改为按原图裁剪区域的像素宽高比显示。
